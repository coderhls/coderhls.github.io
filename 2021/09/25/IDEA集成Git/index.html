2.3 idea来实现git的add操作

当然，也可以选择使用终端来完成操作，但是idea都集成git了，使用起来会比命令行更加方便。 因为git初始化本地库之前pom文件就已经存在，并且还没有添加进git的本地库中，所以这里笔者以这个pom.xml文件为例，右键pom文件，选择git内的add选项，也可以选中pom文件，利用快捷键 ctrl+alt+a快速添加pom到缓存区中

2.4 idea实现commit操作

在上述add操作中，可以看到add上面的一个选项就是commit，对应git命令行中的commit命令

也可以直接点击工具栏中的commit按钮

在这里能够看到add到缓存区的两个文件，在下方的输入框内填写commit的信息

2.5 idea中操作git分支

2.5.1 准备

新建一个Hello类，并提交到本地库

再新增一个输出为 Hello git的语句并提交

2.5.2 打开git面板

在底部找到一个叫git的选项卡，或者快捷键 alt+9打开，在面板中可以看到关于git的所有信息

2.5.3 切换版本

在git面板中可以看到commit提交记录，直接在想要切换的那一个版本右键，选择 checkout revision …即可

可以看到代码内容已经修改 Hello World，没有Hello git，在底下也可以看到切换提示

2.5.4 创建分支

右键项目，在git选项中找到New Branch或在Branches中也可以找到New Branch

同样的，在右下角也有一个Git:master的选项卡，在其中也可以找到创建分支的选项

输入分支名，点击创建即可

在git面板中查看信息的改变 切换回master分支也有很多方式

2.5.5 合并分支

在hot-fix分支中，修改Hello的输出内容为hot-fix

然后提交到本地库中

将hot-fix分支合并到master分支，首先切换到master分支上，右键hot-fix分支选择merge into current

2.5.6 合并分支（解决冲突）

在hot-fix分支新增一个输出 hello python并提交，在master新增一个输出 hello Java提交 将hot-fix分支合并到master分支上，此时出现一个弹窗。idea提示我们需要手动合并分支冲突

选择需要添加的内容到被合并的内容上
