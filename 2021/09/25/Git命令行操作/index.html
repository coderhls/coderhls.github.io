Git命令行操作

发表于2021-09-25|更新于2021-09-25|Git
|阅读量:

1. 本地库初始化

注意：.git目录中存放的是本地库相关的子目录和文件，不要删除，也不要胡乱修改

2. 设置签名

用于区分不同开发人员的身份

项目级别/仓库级别：仅在当前本地库范围内有效

  • git config user.name “your name”
  • git config user.email “your email”

系统级别用户：登陆当前操作系统的用户范围

  • git config –global user.name “your name”
  • git config –global user.email “your email”

级别优先级

  • 就近原则：项目级别优先于系统用户级别，二者都有时采用项目级别
  • 如果只有系统用户级别的签名，则就以系统用户级别的签名为准
  • 二者都没有不允许

3. 常用命令

查看状态 git status

添加操作 git add [file name]

提交操作 git commit -m "commit message" [file name]

查看日志命令

基于索引值进行版本前进后退 git reset --hard [索引值]

reset有三个参数

  • --soft：仅在本地库移动HEAD指针
  • --mixed：在本地库移动HEAD指针，重置暂存区
  • --hard：在本地库移动HEAD指针，重置暂存区，重置工作区

删除文件后找回

比较文件

  • git diff [file name] 将工作区中的文件和暂存区进行比较
  • git diff [本地库中的历史版本] [文件名] 将工作区中的文件和本地库历史记录进行比较

4. 分支操作

使用 git status 可以查看当前分支

使用 git branch -v 可以查看所有分支

创建分支 git branch "分支名"

切换分支 git checkout "分支名"

分支提交操作

合并分支操作

  1. 切换到接受修改的分支（被合并，增加新内容）上 git checkout "分支名"
  2. 执行合并命令 git merge "分支名"
